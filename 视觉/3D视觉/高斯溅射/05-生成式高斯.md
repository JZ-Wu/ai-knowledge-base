# 05 · 生成式高斯

把 3DGS 接进**文生 3D / 图生 3D**——是 2024 年 GS 最热的应用方向。这一篇讲为什么 GS 比 NeRF 更适合做 3D 生成、两条路线（SDS 蒸馏 vs 大模型直出）的差异，DreamGaussian / LGM / GS-LRM 的关键 trick，以及 2024 年底把"生成器"和"3D 表示"解耦的 **TRELLIS**。

> **核心论文**:
> - DreamGaussian (Tang et al., ICLR 2024)
> - LGM (Tang et al., ECCV 2024)
> - GaussianDreamer (Yi et al., CVPR 2024)
> - GRM (Xu et al., ECCV 2024)
> - GS-LRM (Zhang et al., ECCV 2024)
> - **TRELLIS** (Xiang et al., Microsoft, arXiv 2412.01506, 2024-12)

---

## 一、为什么 GS 比 NeRF 适合生成

文生 3D 任务的一个老问题：**慢**。NeRF 时代用 SDS（Score Distillation Sampling）从 2D diffusion 蒸馏 3D 场景，DreamFusion 这种典型方法要训练 1-2 小时才能生成一个 3D 物体。

3DGS 改变了这件事：

1. **渲染快**：每次 SDS 迭代要渲染当前 3D 表示，3DGS 的渲染比 NeRF 快两个数量级，单次 SDS 迭代时间从几秒降到几十毫秒
2. **显式 + 可编辑**：生成后可以直接对高斯做编辑（删/移/复制），NeRF 只能改 MLP 权重
3. **几何与外观解耦**：高斯位置可以由稀疏点云初始化提供几何先验，外观靠 SH/MLP 学
4. **可微 rasterizer 兼容性好**：能直接接进 Stable Diffusion / SDXL 的梯度流

## 二、两条主流路线

| 路线 | 怎么做 | 时间 | 质量 | 代表 |
|------|-------|------|------|------|
| **A. SDS 蒸馏 + GS** | 把 3DGS 当作 SDS 的载体，用 2D diffusion 反传梯度 | 分钟级 | 中等，多视角一致 | DreamGaussian、GaussianDreamer |
| **B. 大模型直出 GS** | 训练一个 transformer/ViT，输入图像，直接输出几百万高斯参数 | < 1 秒 | 高，但受训练数据分布限制 | LGM、GRM、GS-LRM |

A 路线像炼丹（每次都要"训练"一个 3D），B 路线像翻译（前向一次出 3D）。

## 三、DreamGaussian（A 路线代表）

### 3.1 流程：两阶段

**阶段 1：SDS-GS（粗几何）**

- 初始化：少量随机高斯
- 损失：Score Distillation Sampling，让渲染图像看起来像 Stable Diffusion 的样本
  $$\nabla_\theta \mathcal{L}_{\text{SDS}} = \mathbb{E}_{t,\epsilon} [w(t)(\epsilon_\phi(\mathbf{z}_t; t, y) - \epsilon) \frac{\partial \mathbf{z}}{\partial \theta}]$$
- 训练 500-1000 步，约 1-2 分钟拿到粗略 3D 高斯

**阶段 2：Mesh + 纹理优化**

- 从高斯抽 mesh（参考 [02 篇](02-2DGS与表面重建.md)的方法）
- 重新跑 UV mapping + 纹理优化
- 目的：获得游戏引擎可用的 mesh + 纹理

整个过程 **~2 分钟**，生成质量在文生 3D benchmark 上接近 DreamFusion 但快 100 倍。

### 3.2 关键工程 trick

- **零阶 SH**：阶段 1 只用 SH 0 阶（纯漫反射），降参数加速
- **阶段 1 高斯数控制**：限制最多 5000 高斯，避免过拟合 + 加速
- **CFG warmup**：classifier-free guidance scale 从 1 → 30 渐进，避免初期模式崩塌

### 3.3 局限

- SDS 的 multi-face Janus 问题（多视角下出现多张脸）仍存在
- 复杂物体细节不如 stable-zero123 / NeRF-based 方法
- 阶段 2 的 mesh 提取仍然粗糙

---

## 四、GaussianDreamer：3D diffusion 启动 + SDS 精修

DreamGaussian 从随机高斯起步，初期需要大量迭代解决"位置初始化"问题。GaussianDreamer 改进：

### 4.1 流程

1. **3D diffusion 启动**：用 Shap-E / Point-E 这类 3D 扩散模型从文本生成稀疏点云（粗几何，~分钟级）
2. **点云 → 高斯**：每个点初始化一个高斯
3. **SDS 精修**：用 SDXL 反传梯度，~5-10 分钟精修

### 4.2 收益

- 几何先验解决了 multi-face Janus
- 收敛更快（少 50% 迭代）
- 复杂结构（多分支、内嵌结构）质量更好

---

## 五、LGM（B 路线代表）：大模型直出

### 5.1 思路

不要再训练每个物体。训练一个**大型重建模型**，输入 4 张稀疏视图，直接前向输出几百万高斯。

### 5.2 架构

```
4 views (256×256) ──▶ Asymmetric U-Net ──▶ Per-Pixel Gaussians
                          │                       │
                          ▼                       ▼
                  Cross-View Attention     1 view × H × W
                          │              = H×W gaussians × 4 views
                          ▼
                  Multi-View Fusion ──▶ ~250K gaussians total
```

每个像素**输出 14 个数**：3 (位置 offset) + 1 (深度) + 3 (尺度) + 4 (旋转) + 1 (α) + 3 (RGB)，构造一个高斯。这等于把 GS 表示**平铺到像素网格**上。

### 5.3 训练

- 数据：Objaverse-XL（800 万 3D 物体）
- 监督：渲染损失（4 views 当输入，剩余视角作 GT）
- 训练规模：百卡 × 几天
- **推理：< 1 秒**

### 5.4 与 SDS 路线的差异

| 维度 | DreamGaussian (SDS) | LGM (大模型) |
|------|---------------------|-------------|
| 推理时间 | 2 min | < 1 sec |
| 多视角输入 | 文本/单图 | 必须 ≥4 视图 |
| 数据依赖 | 仅依赖 2D diffusion | 依赖 3D 数据集（Objaverse） |
| 分布外泛化 | 好（继承 2D diffusion 的世界知识） | 差（受 Objaverse 物体类别限制） |
| 几何精度 | 中等（SDS 难收敛） | 较好（直接监督） |

LGM 的多视图输入通常先用 MVDream / Zero-1-to-3 这类多视图扩散模型从单图生成，组合起来等价于"单图 → 多视图 → 大模型重建"的 pipeline。

---

## 六、GRM 与 GS-LRM：持续精化

LGM 之后，**GRM**（Gaussian Reconstruction Model）和 **GS-LRM** 在架构上做了几件事：

- **Transformer 替换 U-Net**：cross-view attention 用纯 transformer，质量更稳
- **Per-pixel 改 patch-level**：每个 patch（16×16）输出固定数量的高斯，参数效率提升
- **DUSt3R 启动**：用 DUSt3R 这种立体 3D 重建模型给位置先验，进一步降低对训练数据的依赖
- **更大数据 + 更多视图**：8-16 views 输入，质量明显超过 LGM

到 2024 年下半年，GS-LRM 在物体级 single-image-to-3D 上已经接近"商业可用"水平。

---

## 七、TRELLIS：结构化潜变量 + 多格式解码

> **核心论文**：TRELLIS — *Structured 3D Latents for Scalable and Versatile 3D Generation* (Xiang et al., Microsoft, arXiv 2412.01506, 2024-12)

LGM / GRM / GS-LRM 这条线虽然把推理速度做到了 < 1 秒，但有两个根本约束没解：

- **表示锁死**：模型输出就是高斯参数，想要 mesh 就得另起炉灶训练一个 mesh-LRM
- **几何质量受限**：把高斯"平铺到像素"或"绑到 patch"以方便 CNN/Transformer 输出，复杂几何下表达力不够

TRELLIS 的回答是：**别让模型直接学高斯，让它学一个统一的"结构化 3D 潜变量"，再让多个 decoder 把潜变量翻译成 GS / RF / Mesh**。这等价于把 Stable Diffusion 的"latent space + VAE decoder"思路搬到 3D。

### 7.1 SLAT：稀疏体素 + 稠密局部特征

SLAT (Structured LATents) 由两部分组成：

```
1. 稀疏 3D 结构 (sparse structure)
   ┌─────────────────────────┐
   │ 64³ 体素网格            │
   │ 仅 ~3万 个体素是 active │  ← 决定物体的 occupancy
   └─────────────────────────┘

2. 每个 active 体素挂一个稠密特征向量 (latent)
   z_v ∈ R^d , d ≈ 1024
   特征由 DINOv2 多视图回投得到
```

潜变量的构造方式（编码端）：

1. 给一个 3D 资产渲染 ~150 个视角的图像
2. 每张图过 DINOv2 / DINOv2-Reg 提 patch 特征
3. 把 patch 特征**沿光线反投到 3D 体素**——每个体素聚合所有看到它的视角的特征均值
4. 经过一个 sparse VAE encoder 压成 z_v

**关键点**：SLAT 的潜变量本身**没有几何含义**——它编码的是"在这个位置应该长什么"的语义，而不是"位置在哪、表面法线如何"。几何信息已经被稀疏体素结构 (which voxels exist) 显式分离出来了。这就是"结构化"二字的来源——结构 (sparse mask) 和外观 (per-voxel latent) 被解耦。

### 7.2 两阶段 Flow Matching 扩散

生成时按这个解耦关系做两次扩散：

```
        条件 (image / text embedding)
              │
              ▼
   ┌──── Stage 1 ────┐
   │  Sparse Structure │
   │   Flow Model      │  64³ 全网格上做 flow matching
   │   预测 active mask│  输出 ~30k 个 active voxel 位置
   └────────┬──────────┘
            │
            ▼
   ┌──── Stage 2 ────┐
   │  Structured Latent│
   │   Flow Model      │  仅在 active voxels 上做 sparse transformer
   │   预测 per-voxel z│  输出每个体素的 d 维潜变量
   └────────┬──────────┘
            │
            ▼
       SLAT representation
            │
   ┌────────┼────────┐
   ▼        ▼        ▼
 GS Dec   RF Dec   Mesh Dec
            │
            ▼
   3D Gaussians / Strivec / Mesh
```

**Stage 1 用 dense flow matching**：像素级处理 64³ = 262k 个体素位置，但因为内容稀疏（活的只有 30k 左右），实际是个 binary occupancy 任务。
**Stage 2 用 sparse transformer**：只在 active 体素上做注意力，参数量大但计算量被稀疏度压住，FLOPs 控制在可承受范围。

为什么是 flow matching 而不是 DDPM：训练稳定性显著更好，采样步数少（~25 步即可），符合 [生成模型/Flow Matching 详解](../../生成模型/Flow_Matching详解.md)那条线。

### 7.3 多格式 Decoder 家族

同一个 SLAT 训练**三个独立 decoder**：

| Decoder | 输出 | 工程要点 |
|---------|------|---------|
| **3DGS Decoder** | 每个 active voxel 解码出 K 个高斯（位置 offset + scale + rot + α + SH） | 渲染质量最高，部署友好 |
| **Radiance Field Decoder** | Strivec 风格的 sparse RF | 视觉质量与 GS 接近，但适合体积效应（半透明、毛发） |
| **Mesh Decoder** | Flexicubes 拓扑 + 顶点位置 | 直接给 DCC 工具用，可做拓扑编辑 |

三个 decoder 共享一个 SLAT 空间，意味着**同一个生成结果可以同时拿到三种格式**——这是 LGM/GS-LRM 做不到的。生产里的常见用法：用 GS 渲染做内容审核，用 mesh 进 Blender / Unity，用 RF 做半透明特效。

### 7.4 训练规模与数据

- **数据**：Objaverse-XL（过滤后）+ ABO + 3D-FUTURE + HSSD，~500K 高质量 3D 资产
- **过滤策略**：用 CLIP / 美学评分剔除低质或纹理糟糕的，保留 ~150 视角能渲染稳定的
- **模型规模**：发布两个版本，**TRELLIS-1.1B**（约 11 亿参数）和 **TRELLIS-2B**（约 20 亿）
- **训练算力**：百卡级 × 数周

### 7.5 表现与影响

- **几何 IoU / Chamfer**：在 Objaverse 测试集上比 LGM / GRM / GS-LRM 显著提升（IoU +15-20%，CD 降一半）
- **文本对齐**：CLIP score 接近 SDS 路线（DreamGaussian），但速度快 10×
- **泛化**：分布外物体（动物、人物、室内场景片段）质量明显好于 LGM
- **推理时间**：~30 秒（比 LGM < 1s 慢，但远快于 SDS 路线 2 min）
- **开源**：代码 + 1.1B 权重 2024-12 公开后迅速成为 2025 年 3D 生成的事实基线，很多后续工作直接基于 SLAT 表示做 fine-tune

### 7.6 与前辈对比

| 维度 | LGM | GS-LRM | **TRELLIS** |
|------|-----|--------|-------------|
| 表示 | per-pixel GS | per-patch GS | sparse voxel + 稠密 latent |
| 输出格式 | 仅 GS | 仅 GS | GS / RF / Mesh 三选 |
| 输入 | 4 视图 | 4-16 视图 | 单图 / 多图 / 文本 |
| 生成范式 | 前向回归 | 前向回归 | flow matching diffusion |
| 推理时间 | < 1 s | < 1 s | ~ 30 s |
| 几何质量 | 中等 | 较好 | **显著更好** |
| 开源生态 | 早期 | 学术 | 工业级 |

### 7.7 局限

- **稀疏结构分辨率受限**：64³ 网格在小物体细节上仍不够，2025 年后续工作（如 Direct3D-S2、Hunyuan3D-2）把它推到 256³ + 多尺度
- **推理慢**：30 秒在 UI 交互场景仍是瓶颈，加速版本（distilled / consistency）陆续出现
- **场景级仍弱**：训练数据是物体级，对房间级、场景级生成能力有限，需要后处理拼接
- **可控性**：相比 SDS 蒸馏可以挂任意 ControlNet，TRELLIS 的条件接口仍偏单一

## 八、视频生成 + 4D 生成

GS 生成的下一步是**4D 生成**——让生成的物体动起来。

- **DreamGaussian4D**：DreamGaussian 加时间维度，用视频扩散模型做 SDS
- **AnimateGaussians**：从单视频学习每帧的形变（结合 [03 篇](03-动态与4DGS.md)的 4DGS）
- **Animate3D / 4DGen**：文本 → 4D 物体直出

整体进展比 3D 生成晚一年左右，主要瓶颈：
- 4D 数据稀缺
- SDS 在视频上极不稳定
- 显存：每帧一组高斯，长视频显存吃紧

## 九、编辑（GaussianEditor 等）

生成不是终点，**编辑**是下一步：

| 工作 | 编辑能力 |
|------|---------|
| **GaussianEditor** | 文本 prompt 引导的局部修改（"把这个杯子换成红色"） |
| **GS-Edit / 3DGS Inpainting** | 删除物体 + 补全背景 |
| **Drag-Your-Gaussian** | 拖拽式形变 |
| **TIP-Editor** | 文本 + 参考图双引导 |

工程关键：用 SAM 之类的 2D 分割定位要编辑的高斯子集，再做局部 SDS 优化。

## 十、面试高频问题

### Q1: SDS 是什么？为什么生成 3D 需要它？

**答**：SDS（Score Distillation Sampling）是从 2D diffusion 蒸馏到 3D 表示的方法：当前 3D 渲染图作为 noisy latent，让 diffusion 估计一个去噪方向，把这个方向反传到 3D 参数。这避开了"训练 3D diffusion 需要 3D 数据"的难题——任何能渲染的 3D 表示都能被 2D 扩散模型蒸馏。GS 的渲染快让 SDS 单步迭代时间降到几十毫秒，使生成时间从小时降到分钟。

### Q2: DreamGaussian 和 LGM 各适合什么？

**答**：
- **DreamGaussian**（SDS 路线）：单文本/图 → 任意物体，不依赖 3D 数据集，分布外泛化好，时间 2 min。适合艺术创作、罕见物体。
- **LGM**（大模型路线）：< 1 秒推理，质量稳定，但泛化受 Objaverse 训练数据约束。适合电商、游戏资产快速生成。

### Q3: 为什么 LGM 把高斯"平铺到像素"？

**答**：让 U-Net 的输出维度固定且和图像对齐——每个像素出 1 个高斯，输出 shape 是 (H, W, 14)，标准 CNN 就能处理。如果让网络输出"几百万个无序高斯"，训练目标和监督会困难得多（点云回归是个出名的难问题）。代价是高斯的空间分布被像素网格约束，复杂几何下表达力受限。

### Q4: 多视角 Janus 问题是什么？怎么缓解？

**答**：SDS 训练时不同视角看到的图像各自像 2D diffusion 的 prompt（"a horse"），但缺乏视角一致性约束，导致前后左右都长出脸/眼睛。缓解方法：(1) 几何先验启动（GaussianDreamer 用 Shap-E）；(2) 多视图 diffusion（MVDream / Zero-1-to-3 直接预测一致的多视图）；(3) 加 perceptual 一致性损失。

### Q5: 4D 生成为什么比 3D 生成困难得多？

**答**：(1) 数据稀缺——3D 资产有 Objaverse，但 4D 资产几乎没有；(2) 视频扩散模型本身不稳定，SDS 在动态信号上更难收敛；(3) 表示复杂——动态 GS 参数量上一个量级（[03 篇](03-动态与4DGS.md)）；(4) 评估困难——3D 生成有 multi-view consistency 等指标，4D 加了时间一致性维度。

### Q6: TRELLIS 比 LGM / GS-LRM 强在哪？为什么慢一个量级还是被广泛采用？

**答**：核心改进是**把 3D 生成模型和输出表示解耦**——LGM/GS-LRM 直接学高斯参数，TRELLIS 学一个统一的 SLAT 潜变量（稀疏体素 + 稠密 DINOv2 特征），再训三个独立 decoder 解码到 GS / RF / Mesh。这带来：(1) 同一次推理可同时拿到三种格式，生产管线友好；(2) 几何质量显著上一个台阶（IoU +15%+），因为 latent 空间不被高斯参数化约束；(3) flow matching 训练稳定性好，泛化超过受 Objaverse 训练的 LGM。慢一个量级（30 s vs <1 s）但仍被采用，是因为 (a) 30 s 在内容生产场景可以接受，(b) 多格式输出是工程刚需，(c) 开源 + 1.1B/2B 双版本生态成熟，2025 年很多后续工作直接基于 SLAT 潜空间做 fine-tune 或加速。
