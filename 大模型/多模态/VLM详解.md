# VLM 详解 (Vision-Language Model)

视觉语言模型，让 LLM 能"看懂"图像并用自然语言进行**开放式**视觉理解。

## 〇、VLM 到底在解决什么问题？

### 不只是"看图识字"

VLM 解决的核心问题是：**用自然语言进行开放式的视觉理解和推理**。

"看图识字"（OCR/图像分类）只是 VLM 能力的冰山一角。VLM 真正强大之处在于：

| 能力 | 示例 | 传统 CV 能做吗？ |
|------|------|-----------------|
| 图像描述 | "描述这张图" → 生成一段话 | 不能（只能输出类别/框） |
| 视觉问答 | "图中的人在做什么？" → "他在遛狗" | 不能（没有语言理解） |
| 视觉推理 | "这个场景可能发生在什么季节？为什么？" | 完全不能 |
| OCR + 理解 | 拍一张菜单 → "有哪些素食选项？" | OCR 能识字，但不能理解语义 |
| 文档理解 | 分析表格/图表/论文截图 | 不能 |
| 代码生成 | 看 UI 截图 → 生成前端代码 | 不能 |
| 多图推理 | 比较两张图的差异 | 不能 |
| 视觉定位 | "找到图中的红色汽车" → 输出坐标 | 目标检测能做，但不能自然语言交互 |

### 与传统 CV 的本质区别

```
传统 CV (封闭式):
  输入: 图像
  输出: 固定类别 / 边界框 / 分割掩码
  范式: 训练什么就只能做什么，每个任务一个模型

  ImageNet 分类器: 图像 → "金毛犬" (1000 个类别选一个)
  YOLO 目标检测:   图像 → [类别+框, 类别+框, ...]
  Mask R-CNN:      图像 → [类别+框+掩码, ...]

VLM (开放式):
  输入: 图像 + 任意自然语言问题
  输出: 自然语言回答（可以包含坐标、代码、推理过程等）
  范式: 一个模型解决所有视觉理解任务

  VLM: 图像 + "这是什么品种的狗？" → "这是一只金毛寻回犬，大约2岁..."
  VLM: 图像 + "图中有几个人？" → "3个人，两个站着一个坐着"
  VLM: 图像 + "写一首关于这个场景的诗" → "..."
```

### 范式转变的关键

| 维度 | 传统 CV | VLM |
|------|---------|-----|
| **任务定义** | 预先定义好（分类/检测/分割） | 用自然语言任意定义 |
| **输出形式** | 结构化（类别ID、坐标） | 自然语言（可以是任何内容） |
| **类别集合** | 封闭集（训练时固定） | 开放集（任意类别） |
| **交互方式** | 无（输入图像→输出结果） | 多轮对话 |
| **推理能力** | 无 | 有（继承 LLM 的推理能力） |
| **世界知识** | 无 | 有（LLM 预训练的知识） |
| **一个模型多任务** | 不行（每个任务单独训练） | 可以（统一模型） |

> **一句话总结**：传统 CV 是"看到什么说什么"（模式识别），VLM 是"理解看到的东西并能讨论它"（认知理解）。

## 一、核心架构问题

LLM 只能处理文本 token，如何让它理解图像？

**答案：把图像编码成一组"视觉 token"，和文本 token 一起送入 LLM。**

![LLaVA 架构: Vision Encoder + 投影模块 + LLM (Liu et al., 2023)](assets/llava_architecture.png)

> 图源: *Visual Instruction Tuning (LLaVA)*, Figure 1. 图像经 Vision Encoder 编码为视觉特征，经投影模块对齐维度后，与文本 token 一起送入 LLM 统一处理。

## 二、VLM 三大组件

### 2.1 Vision Encoder（视觉编码器）

将图像编码为特征向量序列。

| 常用选择 | 说明 |
|---------|------|
| **CLIP ViT** | 最主流，天然与语言对齐（LLaVA、Qwen-VL 等） |
| **SigLIP ViT** | CLIP 的改进版，sigmoid 损失（PaliGemma、InternVL2） |
| **DINOv2** | 局部特征更好，常与 CLIP 互补使用 |
| **InternViT** | InternVL 自训练的大规模 ViT（6B 参数） |

**输出**：对于 ViT-L/14 + 336px 图像，输出 576 个 patch token（每个 1024 维）。

### 2.2 投影模块（Vision-Language Connector）

将视觉特征从 Vision Encoder 的维度**映射到 LLM 的输入维度**。

| 方法 | 结构 | 代表模型 |
|------|------|---------|
| **线性投影** | 一个或两个 MLP 层 | LLaVA |
| **Q-Former** | 可学习 query + 交叉注意力，压缩视觉 token 数 | BLIP-2、InstructBLIP |
| **Perceiver Resampler** | 类似 Q-Former 的注意力池化 | Flamingo |
| **C-Abstractor** | 卷积 + 注意力 | Honeybee |
| **Pixel Shuffle** | 空间下采样合并相邻 patch | InternVL2 |

**核心权衡**：

```
少量视觉 token（如 Q-Former 压缩到 32 个）
  → 推理快、省显存
  → 但丢失细节

大量视觉 token（如 576 个甚至 2880 个）
  → 保留细节，OCR/文档理解更好
  → 但推理慢、占用 context window
```

### 2.3 LLM（语言模型骨干）

接收视觉 token + 文本 token，进行理解和生成。

| 模型 | 使用的 LLM |
|------|-----------|
| LLaVA-1.5 | Vicuna (LLaMA 微调) |
| Qwen-VL | Qwen |
| InternVL2 | InternLM2 |
| GPT-4V/4o | GPT-4 系列 |
| Claude 3.x | Claude 系列 |

## 三、主流 VLM 架构对比

### LLaVA 系列（最经典、最简洁）

LLaVA 架构简洁直接: 图像 → CLIP ViT-L/14 → 576 个 patch 特征 → 2 层 MLP 投影 → 视觉 token + 文本 token → LLM → 回答（见上方架构图）。

**LLaVA 的训练分两阶段：**

| 阶段 | 数据 | 训练什么 | 目的 |
|------|------|---------|------|
| **Stage 1: 预训练对齐** | 595K 图文描述对 | 只训练投影层（MLP），冻结 ViT 和 LLM | 学会将视觉特征映射到 LLM 空间 |
| **Stage 2: 指令微调** | 665K 视觉指令数据 | 训练投影层 + LLM（ViT 通常仍冻结） | 学会遵循视觉指令 |

> LLaVA 的重要贡献：证明了**简单架构 + 高质量数据**就能做出强 VLM，不需要复杂的 Q-Former。

### BLIP-2（Q-Former 架构）

### InternVL2（当前最强开源 VLM 之一）

```
特点:
1. 自训练的超大 ViT（InternViT-6B）
2. 动态分辨率：将高分辨率图切成多个 tile，每个 tile 独立编码
3. Pixel Shuffle 下采样减少 token 数
4. 支持多图、视频输入
```

## 四、关键技术问题

### 4.1 高分辨率处理

低分辨率（如 224px）看不清文字和小物体。主流解决方案：

| 方法 | 原理 | 代表 |
|------|------|------|
| **直接放大** | ViT 支持更大输入（如 336/448px），位置编码插值 | LLaVA-1.5 |
| **动态切片 (Tile)** | 将大图切成多个小 tile，分别编码后拼接 | InternVL2、Qwen-VL2 |
| **多尺度** | 同时处理低分辨率全局 + 高分辨率局部 | Monkey、LLaVA-UHD |

动态切片示例：
```
原始高分辨率图 (1344×896)
    │
    ├── 缩放为全局缩略图 (448×448) → 编码 → 全局 token
    │
    └── 切成 3×2 = 6 个 tile (每个 448×448)
        ├── tile 1 → 编码 → 局部 token
        ├── tile 2 → 编码 → 局部 token
        └── ...

所有 token 拼接后送入 LLM
```

### 4.2 视觉 Token 数量控制

高分辨率 = 更多视觉 token = 占用更多 context window = 更慢

| 策略 | 方法 |
|------|------|
| **Pixel Shuffle** | 将相邻 2×2 patch 的特征合并为 1 个（token 数 /4） |
| **Q-Former 压缩** | 用注意力池化压缩到固定数量 |
| **Token 剪枝** | 动态丢弃不重要的视觉 token |
| **Token 合并** | 合并相似的视觉 token (如 ToMe) |

### 4.3 多图 / 视频理解

处理多张图或视频帧：

```
多图: 每张图独立编码，加上图像分隔符 token
      [<img1> v1...vn </img1>] [<img2> v1...vn </img2>] [文本]

视频: 均匀采样若干帧（如 8~32 帧），每帧编码
      [<frame1> ... </frame1>] [<frame2> ...] ... [文本]
```

挑战：视频帧数多 → 视觉 token 爆炸 → 需要更激进的压缩。

## 五、VLM 训练流程

典型的三阶段训练：

```
阶段 1: 视觉-语言对齐预训练
├── 数据: 大规模图文对（数百万~数千万）
├── 训练: 只训投影层，冻结 ViT + LLM
├── 目标: 让视觉特征进入 LLM 的"语义空间"
└── 类比: "安装翻译器"

阶段 2: 多任务预训练
├── 数据: 多种视觉任务数据（描述、问答、OCR、定位...）
├── 训练: 投影层 + LLM（可选：解冻 ViT）
├── 目标: 学习多种视觉理解能力
└── 类比: "上学学各科"

阶段 3: 指令微调 / 偏好对齐
├── 数据: 高质量视觉指令数据 + 偏好数据
├── 训练: 全模型或 LoRA
├── 目标: 遵循指令、回答更好
└── 类比: "入职培训"
```
