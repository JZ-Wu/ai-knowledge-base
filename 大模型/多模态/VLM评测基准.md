# VLM 评测基准

VLM 主流评测基准、各模型成绩对比与评测榜单汇总。

> **版本说明**：本文内容截至 2026 年 4 月，benchmark 数据和模型成绩会持续更新。

## 〇、为什么需要 Benchmark？

在 VLM 领域，模型能力极其多样——识别物体、阅读文字、理解图表、推理空间关系、抵抗幻觉……单靠 "看 demo" 无法系统评估。Benchmark 的核心价值：

1. **标准化对比**：用统一的题目和评分规则，让不同模型在同一把尺子下比较
2. **能力拆解**：把 "多模态理解" 拆成细粒度维度（OCR、空间、推理等），暴露模型短板
3. **追踪进步**：同一个 benchmark 上分数的提升，直观反映技术迭代效果
4. **指导优化**：哪个维度分低，就在对应数据/训练策略上发力

**但也要注意 benchmark 的局限**：
- 刷分 ≠ 真实好用（benchmark 覆盖不了所有真实场景）
- 数据污染问题（训练集可能泄漏 benchmark 题目）
- 评测方式本身有偏差（多选题 vs 开放问答能测的东西不同）

## 一、通用综合评测

这类 benchmark 不聚焦单一能力，而是**全面覆盖**多个维度，给出模型的"综合体检报告"。

### 1.1 MMBench

| 项目 | 内容 |
|------|------|
| **全称** | MMBench: Is Your Multi-modal Model an All-around Player? |
| **出品** | 清华大学 & 上海 AI Lab (2023) |
| **Motivation** | 早期 benchmark（如 VQAv2）题目单一、维度不够细。需要一个能**系统拆解**模型各项能力的综合评测 |
| **任务形式** | 多选题（A/B/C/D），~3000 题 |
| **评价指标** | Accuracy（准确率） |

**核心设计思路**：

- **20+ 能力维度**：将多模态能力细分为 Coarse Perception（粗粒度感知）、Fine-grained Perception（细粒度感知）、Reasoning（推理）三大类，下设属性识别、空间关系、OCR、逻辑推理等 20+ 子维度
- **CircularEval 策略**：同一道题，把选项顺序打乱测多次，只有每次都答对才算对。这样可以过滤掉模型"蒙对"的情况，评测更鲁棒
- **为什么重要**：目前最广泛使用的 VLM 综合 benchmark 之一，几乎所有 VLM 论文都会报 MMBench 分数

### 1.2 MME

| 项目 | 内容 |
|------|------|
| **全称** | MME: A Comprehensive Evaluation Benchmark for Multimodal LLMs |
| **出品** | 2023 |
| **Motivation** | 需要一个**简单直接**、覆盖面广的评测，用最基础的是非题快速摸底模型能力 |
| **任务形式** | 是非题（Yes/No），共 14 个子任务 |
| **评价指标** | Score（每个子任务满分 200，总分 2800） |

**核心设计思路**：

- **感知 vs 认知 双轨**：
  - **感知（Perception）**：物体存在性、计数、位置、颜色、OCR、海报理解、名人识别、场景理解、地标识别、艺术品理解（10 个子任务）
  - **认知（Cognition）**：常识推理、数值计算、文本翻译、代码推理（4 个子任务）
- **计分方式**：每个子任务包含多道题，每题有一正一反两个 Yes/No 问题，两个都答对才得分。防止模型"全答 Yes"刷分
- **为什么重要**：评测成本极低（都是是非题），出结果快，适合快速对比大量模型

### 1.3 SEED-Bench

| 项目 | 内容 |
|------|------|
| **全称** | SEED-Bench: Benchmarking Multimodal LLMs with Generative Comprehension |
| **出品** | Tencent (2023) |
| **Motivation** | 现有 benchmark 只关注静态图像，缺乏**视频理解**评测。需要同时覆盖图像和视频的综合基准 |
| **任务形式** | 多选题，~19K 题（图像 ~14K + 视频 ~5K） |
| **评价指标** | Accuracy |

**核心设计思路**：

- **12 个评测维度**：场景理解、实例身份、实例属性、实例位置、实例计数、空间关系、实例交互、视觉推理、文字理解 + 3 个视频专属维度（动作识别、动作预测、过程理解）
- **题目来源**：基于真实数据集（CC3M、Something-Something 等）+ 人工标注 + GPT 辅助生成
- **为什么重要**：少数同时覆盖图像和视频的综合 benchmark，数据量大，维度全

### 1.4 MM-Vet

| 项目 | 内容 |
|------|------|
| **全称** | MM-Vet: Evaluating Large Multimodal Models for Integrated Capabilities |
| **出品** | NUS (2023) |
| **Motivation** | 多选题/是非题太简单，无法反映模型在**开放式场景**下的真实表现。需要评测模型"综合运用多种能力"解决复杂问题的水平 |
| **任务形式** | 开放式问答（自由回答），218 道精选题 |
| **评价指标** | GPT-4 评分（0-1 分） |

**核心设计思路**：

- **6 大核心能力**：识别（Recognition）、知识（Knowledge）、OCR、空间感知（Spatial Awareness）、语言生成（Language Generation）、数学（Math）
- **能力组合评测**：每道题可能同时需要 2-3 种能力的组合（如"读图中的数字 + 做数学计算"），更贴近真实使用场景
- **GPT-4 自动评分**：开放式回答没有标准答案，用 GPT-4 对模型的回答打分
- **为什么重要**：题目质量高、区分度强，但依赖 GPT-4 评分带来一定偏差（GPT-4 可能偏好自己风格的回答）

### 1.5 MMMU / MMMU-Pro — 大学级多学科多模态理解

| 项目 | 内容 |
|------|------|
| **全称** | MMMU: A Massive Multi-discipline Multimodal Understanding and Reasoning Benchmark |
| **出品** | IN.AI Research & 多所高校 (2024) |
| **Motivation** | 现有 benchmark 大多停留在"常识级"，缺乏**专业领域知识**的评测。大学课程中大量使用图表、公式、电路图等，模型需要具备领域专业知识才能作答 |
| **任务形式** | 多选题 + 开放式问答，~11.5K 题（覆盖 30+ 学科） |
| **评价指标** | Accuracy |

**核心设计思路**：

- **30+ 学科覆盖**：涵盖艺术设计、商科、理科、工科、医学、人文社科六大领域，包含会计、建筑、化学、电子工程、医学影像等具体学科
- **大学考试级难度**：题目来源于真实的大学课程教材、考试题和教科书
- **MMMU-Pro（2025 升级版）**：在原版基础上增加了更多干扰选项（从 4 选 1 变为 10 选 1）、引入 Vision-only 输入（去掉文字提示，纯靠图片作答）、增加开放式问答，难度大幅提升
- **为什么重要**：2025-2026 年最具影响力的 VLM benchmark 之一，顶级模型在 MMMU-Pro 上准确率仍低于 70%，极具区分度

### 1.6 MMStar — 精选高质量综合评测

| 项目 | 内容 |
|------|------|
| **全称** | MMStar: Are We on the Right Way for Evaluating Large Vision-Language Models? |
| **出品** | 2024 |
| **Motivation** | 现有 benchmark 存在严重的**数据泄漏**和**视觉无关性**问题——部分题目不看图就能答对。需要一个"每道题都真正需要视觉理解"的高质量基准 |
| **任务形式** | 多选题，1500 道精选题 |
| **评价指标** | Accuracy |

**核心设计思路**：

- **双重筛选机制**：先用纯文本 LLM 过滤掉"不看图也能答"的题目，再人工核验确保每道题都**视觉不可或缺（Vision-Indispensable）**
- **6 大核心能力 × 18 细分维度**：粗粒度感知、细粒度感知、实例推理、逻辑推理、科学技术、数学
- **为什么重要**：有效避免了数据污染导致的分数虚高，评测结果更可靠。已成为 VLM 论文的标配 benchmark

### 1.7 MEGA-Bench — 超大规模能力评测

| 项目 | 内容 |
|------|------|
| **全称** | MEGA-Bench: Scaling Multimodal Evaluation to over 500 Real-World Tasks |
| **出品** | 2025 |
| **Motivation** | 单一 benchmark 覆盖的任务类型有限，需要一个**海量真实任务**的综合评测来全面刻画模型能力边界 |
| **任务形式** | 500+ 种不同类型的真实世界任务 |
| **评价指标** | 任务特定指标（多种指标聚合） |

**核心设计思路**：

- 覆盖**多图、视频、多轮对话、工具使用**等多种交互形式
- 任务类型极其多样：从基础感知到复杂推理，从日常场景到专业领域
- 支持**能力雷达图**可视化，一目了然看出模型优劣势分布
- **为什么重要**：截至 2026 年规模最大的多模态评测，适合深度分析模型的能力谱系

## 二、专项能力评测

这类 benchmark 聚焦**某一特定能力**，深度测试模型在该维度的上限。

### 2.1 TextVQA — OCR + 阅读理解

| 项目 | 内容 |
|------|------|
| **全称** | Towards VQA Models That Can Read |
| **出品** | Facebook AI (2019) |
| **Motivation** | 早期 VQA 模型几乎完全忽略图像中的文字。一张路标照片问"这是什么路？"，模型答不出来。需要专门测试**读懂图中文字**的能力 |
| **数据** | ~45K 问题，基于 Open Images 中包含文字的图片 |
| **评价指标** | VQA Accuracy（允许多个标准答案，取最高匹配） |

**核心特点**：

- 图片中包含各种文字（路牌、广告牌、书籍封面、运动衫号码等）
- 回答问题必须先"读"到图中的文字，再结合视觉上下文理解
- **与纯 OCR 的区别**：不是简单提取文字，而是需要理解文字的含义和上下文（如"封面上的作者是谁？"）

### 2.2 DocVQA — 文档理解

| 项目 | 内容 |
|------|------|
| **全称** | DocVQA: A Dataset for VQA on Document Images |
| **出品** | CVC Barcelona (2021) |
| **Motivation** | 企业场景大量依赖文档（发票、合同、报告），需要模型能**理解文档版面布局**，从中提取和推理信息 |
| **数据** | ~50K 问题，基于 ~12K 文档图像（行业文档） |
| **评价指标** | ANLS（Average Normalized Levenshtein Similarity），容忍 OCR 的小错误 |

**核心特点**：

- 文档种类丰富：表格、表单、信件、报告、发票等
- 不仅要识别文字，还要理解**版面结构**（标题在哪、表格怎么读、哪些内容是关联的）
- 实际工业应用价值极高——IDP（智能文档处理）的核心评测

### 2.3 ChartQA — 图表理解

| 项目 | 内容 |
|------|------|
| **全称** | ChartQA: A Benchmark for Question Answering about Charts |
| **出品** | 2022 |
| **Motivation** | 报告、论文中大量使用图表传递信息。模型需要理解柱状图、折线图、饼图的**视觉编码**并做数值推理 |
| **数据** | ~32K 问题（人工 + 机器生成），覆盖多种图表类型 |
| **评价指标** | Relaxed Accuracy（数值答案允许 5% 误差） |

**核心特点**：

- 两类问题：**人工标注**（需要复杂推理，如"哪一年增速最快？"）和**机器生成**（较简单，如"2020 年的值是多少？"）
- 需要模型理解图表的视觉映射规则（x 轴是什么、颜色代表什么、柱子高度对应什么值）
- 综合考验 OCR（读坐标轴标签）+ 视觉理解（看柱子/线的趋势）+ 数值推理（比较、计算）

### 2.4 MathVista — 数学视觉推理

| 项目 | 内容 |
|------|------|
| **全称** | MathVista: Evaluating Mathematical Reasoning in Visual Contexts |
| **出品** | UCLA & Microsoft (2023) |
| **Motivation** | 数学推理和视觉理解是两个独立研究方向，但真实场景中经常需要**看图做数学题**（几何图形、函数图像、统计图表）。缺乏专门评测这种交叉能力的 benchmark |
| **数据** | 6141 题，来自 31 个已有数据集的汇编 |
| **评价指标** | Accuracy |

**核心特点**：

- **5 类数学推理**：代数、算术、几何、逻辑、统计
- **7 类视觉上下文**：几何图形、函数图、表格、柱状图、折线图、地图、科学图
- 难度跨度大：从小学数学到大学级别
- 模型在此 benchmark 上的分数通常偏低，暴露了当前 VLM 在**精确数学推理**上的短板

### 2.5 RealWorldQA — 真实场景理解

| 项目 | 内容 |
|------|------|
| **出品** | xAI (2024) |
| **Motivation** | 许多 benchmark 用的是"干净"的图片，而现实世界照片往往有复杂背景、光照变化、遮挡等。需要测试模型在**真实拍摄场景**下的理解能力 |
| **数据** | ~700 张真实世界照片 + 多选题 |
| **评价指标** | Accuracy |

**核心特点**：

- 图片来自真实拍摄（不是网络爬取的清晰图片），包含驾驶场景、日常生活场景等
- 问题侧重**实用性**（如"这个路口可以左转吗？""这个停车位能停吗？"）
- 数据量较小，但题目质量高，区分度好

### 2.6 AI2D — 科学图表理解

| 项目 | 内容 |
|------|------|
| **全称** | AI2D: A Dataset of Illustrative Diagrams for Research on Diagram Understanding |
| **出品** | Allen AI (AI2) |
| **Motivation** | 科学教育中大量使用示意图（食物链、细胞结构、水循环等），模型需要理解图中**元素之间的关系和标注** |
| **数据** | ~5000 张科学图 + ~15K 多选题 |
| **评价指标** | Accuracy |

**核心特点**：

- 图片都是科学教育领域的示意图/流程图
- 需要理解箭头指向、标注含义、元素间的因果/层级关系
- 与 ChartQA 的区别：ChartQA 侧重**数据图表**（统计数据），AI2D 侧重**概念示意图**（知识结构）

### 2.7 POPE — 物体幻觉评测

| 项目 | 内容 |
|------|------|
| **全称** | Evaluating Object Hallucination in Large Vision-Language Models (POPE) |
| **出品** | 2023 |
| **Motivation** | VLM 最大的痛点之一是**幻觉**——图中没有的东西，模型"看到了"。例如图中没有猫，模型却说"图中有一只猫"。需要专门量化这个问题 |
| **任务形式** | 是非题——"图中是否有 [某物体]？" |
| **评价指标** | Accuracy、F1-Score、Yes 比例 |

**核心设计思路**：

- **三种采样策略**（难度递增）：
  - **Random**：随机选不存在的物体问"有没有"
  - **Popular**：选数据集中**高频出现**的物体（模型更容易误判为有）
  - **Adversarial**：选与图中已有物体**高共现**的物体（如图中有"餐桌"，问"有没有椅子"——因为餐桌和椅子经常一起出现，模型极易幻觉）
- **Yes 比例监控**：如果模型 Yes 比例远高于 50%，说明有严重的 Yes-bias（倾向于说"有"）
- **为什么重要**：幻觉是 VLM 实际部署中最影响用户信任的问题，POPE 是最标准的量化工具

### 2.8 HallusionBench — 综合幻觉评测

| 项目 | 内容 |
|------|------|
| **全称** | HallusionBench: An Advanced Diagnostic Suite for Entangled Language Hallucination and Visual Illusion in LVLMs |
| **出品** | 2024 |
| **Motivation** | POPE 只测物体级幻觉，实际幻觉类型更多样——属性错误、关系错误、视觉错觉等。需要更**全面系统**的幻觉评测 |
| **数据** | 346 张图 + 1129 个问题 |
| **评价指标** | Accuracy（整体 + 分维度） |

**核心特点**：

- 区分两类幻觉：
  - **语言幻觉（Language Hallucination）**：模型被文字误导，而非真正"看"图
  - **视觉错觉（Visual Illusion）**：图片本身具有欺骗性（如视觉错觉图），测试模型是否能正确感知
- 问题更加刁钻：不只是"有没有某物体"，而是涉及数量、颜色、空间位置等多种属性的幻觉
- 与 POPE 互补，共同构成幻觉评测的完整拼图

### 2.9 BLINK — 人易模型难的视觉感知

| 项目 | 内容 |
|------|------|
| **全称** | BLINK: Multimodal Large Language Models Can See but Not Perceive |
| **出品** | 2024 |
| **Motivation** | 许多对人类来说"一眼就能看出"的视觉判断（如深度比较、拼图匹配、相对大小），VLM 却频繁出错。需要专门暴露模型**底层视觉感知**的缺陷 |
| **数据** | 3807 道多选题，涵盖 14 种视觉感知任务 |
| **评价指标** | Accuracy |

**核心特点**：

- **14 种感知任务**：相对深度、拼图还原、视觉对应、语义对应、多视角推理、视觉相似度、艺术风格识别、功能对应等
- 人类准确率 >95%，但最强 VLM 也仅 ~60%，**差距巨大**
- 揭示了 VLM"能理解但不能感知"的根本性短板——语言推理强但视觉基础弱

### 2.10 MMLongBench-Doc — 长文档理解

| 项目 | 内容 |
|------|------|
| **全称** | MMLongBench-Doc: Benchmarking Long-context Document Understanding |
| **出品** | 2025 |
| **Motivation** | 真实企业场景中的文档往往长达数十页甚至上百页（年报、合同、手册），现有 DocVQA 只测单页文档，远不够 |
| **数据** | 1062 个问题，基于 130 篇长文档（平均 47 页） |
| **评价指标** | F1-Score |

**核心特点**：

- 文档长度 **10-120 页**，远超单页 DocVQA
- 需要跨页面检索、关联和推理——"第 3 页的表格数据与第 15 页的结论是否一致？"
- 测试模型的**长上下文多模态理解**能力，对上下文窗口提出极高要求
- 与 2025-2026 年长上下文模型（百万 token 级）的发展趋势高度匹配

### 2.11 MMT-Bench — 多任务多模态评测

| 项目 | 内容 |
|------|------|
| **全称** | MMT-Bench: A Comprehensive Multimodal Benchmark for Evaluating LMMs on Massive and Diverse Tasks |
| **出品** | 2024 |
| **Motivation** | 评测覆盖的任务类型越多，越能发现模型的"长尾弱点"。需要一个**大规模、高多样性**的任务集 |
| **数据** | ~32K 题，覆盖 162 种子任务 |
| **评价指标** | Accuracy |

**核心特点**：

- 涵盖视觉识别、定位、OCR、计数、3D 感知、动作理解等 30+ 能力维度
- 引入了**自动驾驶、机器人、医疗影像**等垂直行业场景
- 适合对模型进行"全景扫描"式评测

## 三、视频理解评测

视频评测相比图像评测增加了**时间维度**——模型需要理解动作、事件顺序、因果关系等时序信息。

### 3.1 MVBench

| 项目 | 内容 |
|------|------|
| **全称** | MVBench: A Comprehensive Multi-modal Video Understanding Benchmark |
| **出品** | 2024 |
| **Motivation** | 视频理解维度复杂，需要一个**细粒度拆解时序能力**的 benchmark |
| **任务形式** | 多选题，20 个时序理解维度 |
| **评价指标** | Accuracy |

**20 个维度包括**：动作序列、动作预测、动作反推、动作定位、动作计数、异常检测、物体存在性、角色识别、场景切换、移动方向、运动属性 等。

**核心特点**：
- 每个维度独立评分，可以精准定位模型在哪类时序推理上弱
- 从 11 个已有视频数据集中汇编，确保多样性

### 3.2 Video-MME

| 项目 | 内容 |
|------|------|
| **全称** | Video-MME: The First-Ever Comprehensive Evaluation Benchmark of Multi-modal LLMs in Video Analysis |
| **出品** | 2024 |
| **Motivation** | 现有视频 benchmark 主要用短视频片段，但实际应用中需要处理**不同时长**的视频。需要系统评测短/中/长视频理解能力 |
| **数据** | 900 个视频，2700 个问题 |
| **评价指标** | Accuracy |

**核心设计思路**：

- **三档时长**：
  - 短视频（<2 min）
  - 中视频（4-15 min）
  - 长视频（30-60 min）
- **有/无字幕两种设置**：测试模型是否依赖字幕理解内容
- 涵盖 30+ 视频类型（电影、教育、体育、游戏、监控等）
- **为什么重要**：长视频理解是当前 VLM 的前沿难题（上下文窗口、信息压缩、关键帧选取），这个 benchmark 直接量化这个能力

### 3.3 EgoSchema

| 项目 | 内容 |
|------|------|
| **全称** | EgoSchema: A Diagnostic Benchmark for Very Long-form Video Language Understanding |
| **出品** | 2023 |
| **Motivation** | 第一人称视频（如 AR 眼镜录像）是重要的应用场景，但模型在这类视频上的**长程时序推理**能力未被充分评测 |
| **数据** | 5031 道多选题，基于 Ego4D 数据集（第一人称活动视频） |
| **评价指标** | Accuracy |

**核心特点**：

- 每个视频长达 **3 分钟**（相比传统视频 QA 的几秒钟，极具挑战）
- 问题需要理解整个视频的长程上下文，而非某一帧
- 第一人称视角带来额外难度：视角不稳定、遮挡多、自我运动干扰
- 子集 EgoSchema-500 常用于快速评测

### 4.4 从成绩中读出的趋势 (2025-2026)

- **开源全面追平闭源**：Qwen2.5-VL-72B、InternVL3 在多项 benchmark 上达到甚至超过 GPT-4.1 和 Claude 4 Sonnet，开闭源差距基本消失
- **Gemini 2.5 Pro 多项领先**：Google 在多模态方向持续发力，Gemini 2.5 Pro 在 DocVQA、ChartQA、长文档等场景表现突出
- **推理模型崛起**：o3 等推理模型在 MathVista、MMMU-Pro 等需要深度推理的 benchmark 上显著领先传统模型
- **数学推理显著进步**：MathVista 从 2024 年的 ~63 提升到 2026 年的 ~75，但 MMMU-Pro 仍是所有模型的难点（<60%）
- **底层视觉感知仍是短板**：BLINK 上最强模型仍只有 ~62%，远低于人类 >95%，说明"能理解但不能精确感知"的根本问题尚未解决
- **长文档是新前沿**：MMLongBench-Doc 上所有模型均低于 55%，长上下文多模态理解仍有巨大提升空间
- **中小尺寸模型快速进步**：7B-13B 级模型（如 Qwen2.5-VL-7B、InternVL3-8B）在 2026 年已能达到 2024 年 70B 模型的水平

## 六、如何选择 Benchmark？

不同场景关注不同 benchmark：

| 你的目标 | 优先看这些 Benchmark |
|---------|---------------------|
| 全面了解模型综合能力 | MMBench、MMStar、MEGA-Bench |
| 评估专业/学术场景 | MMMU、MMMU-Pro |
| 评估文档处理/OCR 场景 | DocVQA、TextVQA、ChartQA、MMLongBench-Doc |
| 评估模型可靠性（幻觉） | POPE、HallusionBench |
| 评估数学/科学推理 | MathVista、AI2D |
| 评估底层视觉感知 | BLINK |
| 评估视频理解 | Video-MME、MVBench、EgoSchema |
| 了解真实用户体验 | LMSYS Vision Arena、WildVision |
| 选型决策（性能+成本） | Artificial Analysis + Arena 排名 |

> **建议**：不要只看一个 benchmark 的分数，要**综合多个 benchmark** 形成全面判断。同时关注 Arena 排名（人类偏好）和固定题目 benchmark（客观分数）的差异。2025 年以来，推荐优先参考 **MMMU-Pro + MMStar + BLINK** 这组新 benchmark——它们在数据质量和区分度上优于早期基准。
