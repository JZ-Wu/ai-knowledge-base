# 3D 骨干网络 论文阅读

> 整理自 2026 年 1 月周报 | 更新日期: 2026-03-15

---

## Point Transformer (2020)

> Zhao et al., Oxford & HKU | ICCV 2021 | https://arxiv.org/abs/2012.09164

首次将 Transformer 架构成功应用于点云处理。

### 核心算子：Point Transformer Layer

$$y_i = \sum_{x_j \in \text{kNN}(i)} \rho \Big( \gamma(\varphi(x_i) - \psi(x_j) + \delta) \Big) \odot \Big( \alpha(x_j) + \delta \Big)$$

关键设计：
- **局部性**: 只计算 k 近邻 (kNN)
- **减法关系**: 使用 $\varphi(x_i) - \psi(x_j)$ 而非点积，在几何空间中差值比夹角更能描述相对关系
- **双重位置注入**: $\delta = \theta(p_i - p_j)$ 同时加在注意力生成和特征变换中

### 向量注意力 vs. 标量注意力

| 特性 | 标量注意力 (Standard) | 向量注意力 (PTv1) |
|------|----------------------|-------------------|
| 权重维度 | 标量 (1D) | 向量 (C-dim) |
| 数学操作 | 标量乘法 (Broadcast) | 哈达玛积 ($\odot$, Element-wise) |
| 通道控制 | 所有通道共用一个权重 | 每个通道独立权重 |
| 物理意义 | 调节"总音量" | 多频段均衡器 (EQ) |

向量注意力让模型可以高亮某通道的几何特征，同时抑制另一通道的噪音。对 3D 点云至关重要，因为几何特征往往高度解耦。

### 网络架构: 3D 版 U-Net

- **下采样 (Transition Down)**: FPS (最远点采样) → kNN → Local Max Pooling
- **上采样 (Transition Up)**: Linear → 三线性插值 → Skip Connection (Add)

## Swin Transformer (2021)

> Liu et al., Microsoft | ICCV 2021 (Best Paper) | https://arxiv.org/abs/2103.14030

解决 Transformer 在高分辨率图像上的计算瓶颈，引入 **"Shifted Window"** 机制。理解 PTv3 的前置知识。

### ViT 的两个致命弱点

1. **直筒子结构**: 输出特征图分辨率始终不变 (1/16)，缺乏多尺度特征金字塔
2. **计算量爆炸**: 全局注意力 $O(N^2)$，大图显存溢出

### W-MSA + SW-MSA (Tick-Tock)

- **W-MSA (Window Attention)**: 切成固定窗口 ($7 \times 7$ Patch)，窗口内部算 Attention → $O(N)$
- **SW-MSA (Shifted Window)**: 下一层窗口位置移动半个窗口尺寸 → 原来的"边界"变成新的"中心"，信息跨窗口流通
- **工程实现**: Cyclic Shift + Mask (把移出边界的像素"卷"回来，用 Mask 阻断不合法交互)

### 4-Stage 金字塔

| Stage | 操作 | 输出分辨率 |
|-------|------|-----------|
| Stage 1 | Patch Partition (4x4) | $H/4, W/4$ |
| Stage 2 | Patch Merging | $H/8, W/8$ |
| Stage 3 | Patch Merging (主力) | $H/16, W/16$ |
| Stage 4 | Patch Merging | $H/32, W/32$ |

Patch Merging: 取 $2 \times 2$ 邻近 Patch 拼接 + Linear 降维，图像高宽减半，通道翻倍。

## Point Transformer V3 (2023)

> Wu et al., HKU | CVPR 2024 | https://arxiv.org/abs/2312.10035
> 代码: https://github.com/Pointcept/PointTransformerV3 (`model.py`, ~983行)

彻底放弃对"几何精细度"的执念，全面拥抱 **Scale**。可理解为 **"3D 点云版的 Swin Transformer"**。

### 演进: 从"精雕细琢"到"大力出奇迹"

| | PTv1/v2 (学术派) | PTv3 (工程派) |
|--|-----------------|---------------|
| 邻域构建 | KNN 搜索 (随机内存访问) | Z-Order 排序 (连续访问) |
| 注意力 | Vector Attention (无法加速) | Dot-Product + FlashAttention (3-10x 加速) |
| 感受野 | 16~32 点 | 1024+ 点 |

---

### 整体流水线 (Forward Pass)

```
输入 data_dict
  │
  ▼
Point(data_dict)          ← 封装为 Point 对象 (coord, feat, offset 等)
  │
  ▼
Point.serialization()     ← 空间填充曲线编码，生成 serialized_order
  │
  ▼
Point.sparsify()          ← 构建 SparseTensor (用于稀疏卷积)
  │
  ▼
Embedding                 ← MLP 将原始特征映射到 enc_channels[0]
  │
  ▼
Encoder (5 stages)        ← 每个 stage: [SerializedPooling] + N × Block
  │                          enc_depths=(2,2,2,6,2), channels=(32,64,128,256,512)
  │                          stride=(2,2,2,2)
  ├──skip──┐
  ▼        │
Decoder (4 stages)        ← 每个 stage: SerializedUnpooling + skip + N × Block
  │
  ▼
输出 Point (带 per-point 特征)
```

**默认配置**: 5 个 encoder stage, 4 个 decoder stage, `patch_size=1024`, `num_heads=(2,4,8,16,32)`。

### 核心一: 空间填充曲线序列化 (`Point.serialization()`)

#### 为什么需要序列化？

PTv1/v2 用 KNN 找邻域 → 随机内存访问 → GPU 利用率极低。PTv3 的核心洞察: **把 3D 空间"展平"成 1D 序列，让空间相近的点在内存中也相邻**，这样直接切片就是近似的局部邻域。

#### 序列化流程

```python
# Step 1: 坐标体素化
grid_coord = (coord / grid_size).floor().int()  # 连续坐标 → 整数网格坐标

# Step 2: 自适应确定编码深度
depth = grid_coord.max().bit_length()  # 最大坐标需要多少位表示
# depth 上限 16，即最大支持 2^16 = 65536 个网格 (grid_size=0.01m 时约 655m)

# Step 3: 空间填充曲线编码（每个点 → 1 个 int64）
code = encode(grid_coord, depth)  # Z-Order 或 Hilbert

# Step 4: 批次编码到高位 (保证不同样本的点不会交叉)
code = batch_index << (depth * 3) | spatial_code
#       ^^^^^^^^^^^^^^^^^^^^^^^^   ^^^^^^^^^^^^
#       高位: 样本编号               低位: 空间位置

# Step 5: argsort 得到排序映射
serialized_order = torch.argsort(code)       # 排序位置 → 原始索引
serialized_inverse = scatter_inverse(order)  # 原始索引 → 排序位置
```

#### Z-Order 编码 (Morton Code)

核心是**位交叉 (Bit Interleaving)**: 将 x, y, z 的二进制位交替排列。

```
x = 5 = 101₂     y = 3 = 011₂     z = 6 = 110₂

位交叉: z₂ y₂ x₂ z₁ y₁ x₁ z₀ y₀ x₀
        1   0   1   1   1   0   0   1   1  → 代码就是 101110011₂
```

**实现优化**: 代码使用 **查找表 (LUT)** 方法（来自 OCNN），预计算 8-bit 值的交叉结果，16-bit 坐标分两次查表 + 移位拼接，比逐位循环快 10 倍+。

#### Hilbert 曲线编码

Hilbert 曲线比 Z-Order 的**空间连续性更好** (没有大跳跃)，但编码更复杂。代码实现了 **Skilling 算法的 PyTorch 向量化版本** (~303行):

```
Z-Order:  ...连续一段... ← 大跳跃 → ...连续一段...
Hilbert:  ...连续一段... → 拐弯 → ...连续一段...  (始终相邻)
```

#### 4 种序列化顺序

```python
order = ["z", "z-trans", "hilbert", "hilbert-trans"]
```

- `z`: 标准 Z-Order (xyz)
- `z-trans`: 转置轴的 Z-Order (zyx)，让不同维度轮流做"优先排序轴"
- `hilbert`: 标准 Hilbert 曲线
- `hilbert-trans`: 转置轴的 Hilbert 曲线

**Shuffle Orders**: 训练时随机打乱这 4 种顺序的使用次序，作为数据增强。

### 核心二: 分块 (Patching) 与不定长处理

#### 分块策略

序列化后，点云变成 1D 有序序列。直接按固定大小切块:

```
排序后的点: [p₀ p₁ p₂ ... p₁₀₂₃ | p₁₀₂₄ p₁₀₂₅ ... p₂₀₄₇ | ...]
             ←── Patch 0 ──→     ←──── Patch 1 ────→
```

每个 Patch 内部做 Self-Attention，不同 Patch 之间不交互 (类似 Swin 的 Window Attention)。

#### 不定长处理 (`get_padding_and_inverse`)

一个 batch 中不同点云的大小不同（如样本A有15000点，样本B有12000点），需要 padding:

```python
def get_padding_and_inverse(offset, patch_size):
    # offset = [15000, 27000]  → 样本A: 15000点, 样本B: 12000点

    # Step 1: 计算每个样本的点数
    counts = [15000, 12000]

    # Step 2: 向上对齐到 patch_size 的倍数
    # 15000 → ceil(15000/1024)*1024 = 15360  (多出 360 个位置)
    # 12000 → ceil(12000/1024)*1024 = 12288  (多出 288 个位置)

    # Step 3: padding 策略 —— 不是填零！而是复制上一个 patch 的点
    # 这样 padding 区域的注意力不会浪费在无意义的 token 上
    pad = [0,1,2,...,14999, 14024,...,14383,   # 样本A: 原始点 + 复制尾部patch的点
           15000,...,26999, 26024,...,26311]     # 样本B: 同理

    # Step 4: 计算反向映射 (从 padded 回到原始索引)
    unpad = ...  # 用于 attention 输出后"去掉"padding

    # Step 5: cu_seqlens (Flash Attention 需要的累积序列长度)
    cu_seqlens = [0, 1024, 2048, ..., 27648]  # 每个 patch 的边界
```

**关键设计**: padding 不用零，而是**复制相邻点**。这样 padding 区域的注意力计算出的结果虽然会被 `unpad` 丢弃，但它们在计算过程中不会稀释真实点的注意力分数（因为 padding 点在空间上与真实点相邻，是合理的邻居）。

#### Flash Attention 的变长接口

```python
# 使用 flash_attn_varlen_qkvpacked_func
# cu_seqlens = [0, 1024, 2048, 3072, ...]  每个patch的起止位置
# max_seqlen = 1024 (patch_size)
output = flash_attn_varlen_qkvpacked_func(
    qkv,                    # (total_tokens, 3, num_heads, head_dim)
    cu_seqlens=cu_seqlens,  # 告诉 Flash Attention 每个"序列"的边界
    max_seqlen=patch_size   # 所有序列的最大长度
)
```

这比自己做 padding + mask 高效得多，因为 Flash Attention 原生支持变长序列。

### 核心三: 位置编码 —— CPE (Conditional Positional Encoding)

#### 为什么不用传统 RPE？

PTv1/v2 在注意力分数中加入 Relative Position Encoding (RPE)。但 RPE 需要显式计算 $K \times K$ 的相对位置矩阵，**与 Flash Attention 不兼容** (Flash Attention 不支持自定义 attention bias)。

#### CPE 的实现

```python
# Block.__init__
self.cpe = PointSequential(
    spconv.SubMConv3d(channels, channels, kernel_size=3, bias=True, indice_key=cpe_indice_key),
    nn.Linear(channels, channels),
    nn.LayerNorm(channels),  # 或 BatchNorm
)

# Block.forward
shortcut = point.feat
point = self.cpe(point)          # 3x3x3 稀疏子流形卷积
point.feat = shortcut + point.feat  # 残差连接
# → 然后进入 LayerNorm → Attention → ...
```

**原理**: 3×3×3 的稀疏子流形卷积 (SubMConv3d) **隐式编码了 3D 空间位置**。因为卷积核的权重是与空间偏移绑定的 (第 $[dx, dy, dz]$ 个位置的邻居用第 $[dx, dy, dz]$ 组权重)，每个点经过卷积后的特征就包含了"我周围的 26 个邻居分别在哪、特征是什么"的信息。

**双重作用**:
1. **位置编码**: 让 Attention 知道点的空间关系
2. **缝合序列化跳跃**: Z-Order/Hilbert 曲线不完美，相邻 Patch 的边界点在 3D 空间中可能不相邻。CPE 的卷积在 **3D 真实空间** 中操作，不受序列化顺序影响，相当于在每个 Block 开头修复一次序列化带来的空间割裂

#### 可选的 RPE (非 Flash Attention 模式)

当 `enable_flash=False` 时，代码支持传统 RPE:

```python
# 计算 patch 内每对点的相对网格坐标
rel_coord = grid_coord[i] - grid_coord[j]  # (num_patches, K, K, 3)

# clamp 到 [-pos_bnd, pos_bnd]，索引查找表
rel_coord = rel_coord.clamp(-pos_bnd, pos_bnd) + pos_bnd
# 查找表: (3*(2*pos_bnd+1), num_heads)，x/y/z 各一组
bias = table[rel_x] + table[rel_y + offset] + table[rel_z + 2*offset]
# → (num_patches, num_heads, K, K)，加到 attention score 上
```

### 核心四: Attention 计算 (`SerializedAttention`)

#### 完整流程

```python
def forward(self, point):
    # 1. 获取当前 block 使用的序列化顺序
    #    连续 block 轮换: block_0 用 z, block_1 用 z-trans, block_2 用 hilbert, ...
    order_index = self.block_index % len(self.order)
    order = point.serialized_order[order_index]      # 排序映射
    inverse = point.serialized_inverse[order_index]  # 逆映射

    # 2. 计算 QKV
    qkv = self.qkv(point.feat)  # (N, 3 * C)
    qkv = qkv.reshape(N, 3, num_heads, head_dim)

    # 3. Padding + 按序列化顺序重排
    pad, unpad, cu_seqlens = get_padding_and_inverse(offset, patch_size)
    qkv = qkv[order[pad]]  # 先排序，再 padding → 得到连续内存的 patched 序列

    # 4. Flash Attention (patch 内)
    out = flash_attn_varlen_qkvpacked_func(
        qkv, cu_seqlens, max_seqlen=patch_size
    )

    # 5. 反向映射回原始顺序
    out = out[unpad][inverse]  # 去 padding → 反排序

    # 6. 输出投影
    point.feat = self.proj(out.reshape(N, C))
    return point
```

#### 多顺序轮换 (Shuffle Order)

这是 PTv3 对 Swin "Shifted Window" 的替代方案:

```
Block 0: Z-Order 切片       →  [P0 P1 P2 P3 | P4 P5 P6 P7 | ...]
Block 1: Z-Order-Trans 切片 →  [P0 P3 P5 P1 | P2 P7 P4 P6 | ...]  ← 不同分组
Block 2: Hilbert 切片       →  [P0 P1 P3 P5 | P2 P4 P6 P7 | ...]  ← 又不同
Block 3: Hilbert-Trans 切片 →  [P5 P3 P1 P0 | P7 P6 P4 P2 | ...]
```

每种曲线产生不同的 1D 排列 → 不同的 Patch 分组 → 不同的点参与同一个 Attention 窗口。效果等价于 Swin 的 Shifted Window: **让不同 Block 看到不同的局部邻域**，信息逐步跨区域流通。

### 核心五: 下采样 (`SerializedPooling`)

```python
def forward(self, point):
    # 1. 右移序列化代码 → 合并体素
    #    stride=2 时, pooling_depth=1, 右移 3 位
    #    → 相当于把 2×2×2 的 8 个体素合并为 1 个
    code = cycled_code >> (pooling_depth * 3)

    # 2. torch.unique 找到唯一的合并后体素
    cluster, inverse = torch.unique(code, return_inverse=True)

    # 3. 特征聚合 (投影 + 池化)
    feat = self.proj(point.feat)  # Linear: C_in → C_out
    feat = segment_csr(feat, ...)  # 按 cluster 做 reduce (sum/mean/max)

    # 4. 重建 Point 对象
    #    - 新的 coord = scatter_mean(old_coord, cluster)
    #    - 新的 grid_coord = grid_coord >> 1  (坐标减半)
    #    - 重新序列化 (因为合并后需要新的排序)
    #    - 重新 sparsify (构建新的 SparseTensor)

    # 5. 保存 pooling_inverse 供 decoder 上采样使用
    point.pooling_inverse = inverse
    return point
```

**位运算的精妙**: Z-Order/Hilbert 编码的每 3 位代表一个空间层级。右移 3 位 = 丢弃最低 1 层的空间分辨率 = 2×2×2 合并。这比重新体素化 + 聚合高效得多。

### 核心六: 上采样 (`SerializedUnpooling`)

```python
def forward(self, point, skip_point):
    # 1. 特征投影
    point.feat = self.proj(point.feat)       # 上采样路径
    skip_point.feat = self.skip_proj(skip_point.feat)  # skip 路径

    # 2. 最近邻上采样: 用 pooling_inverse 把低分辨率特征"复制"回高分辨率
    #    pooling_inverse[i] = j 表示高分辨率点 i 属于低分辨率体素 j
    skip_point.feat = skip_point.feat + point.feat[skip_point.pooling_inverse]

    return skip_point
```

对比 PTv1 的三线性插值: 上采样只用最近邻复制 (无需距离计算)，精度靠 Skip Connection 补回来。

### Block 完整结构

```python
class Block(nn.Module):
    def forward(self, point):
        # 1. CPE: 稀疏卷积位置编码 (残差)
        shortcut = point.feat
        point = self.cpe(point)           # SubMConv3d(3×3×3) + Linear + Norm
        point.feat = shortcut + point.feat

        # 2. Self-Attention (Pre-Norm)
        shortcut = point.feat
        point.feat = self.norm1(point.feat)
        point = self.attn(point)          # SerializedAttention
        point.feat = shortcut + self.drop_path(point.feat)

        # 3. FFN (Pre-Norm)
        shortcut = point.feat
        point.feat = self.norm2(point.feat)
        point.feat = self.mlp(point.feat)  # Linear → GELU → Linear
        point.feat = shortcut + self.drop_path(point.feat)

        # 4. PDNorm (可选，多数据集训练时的条件归一化)
        return point
```

### 与 Swin 的对应关系

| Swin (2D) | PTv3 (3D) | 代码位置 |
|-----------|-----------|---------|
| Window Partition | Patch Grouping (序列化切片) | `get_padding_and_inverse` |
| Shifted Window | Shuffle Order (轮换 4 种曲线) | `SerializedAttention.forward` |
| Relative Position Bias | CPE (3×3×3 稀疏卷积) | `Block.cpe` |
| Patch Merging | SerializedPooling (位运算合并) | `SerializedPooling` |
| Patch Expanding | SerializedUnpooling (最近邻+skip) | `SerializedUnpooling` |

### 为什么 PTv3 能赢？

1. **Don't fight the hardware**: 用序列化把数据排整齐，适配 FlashAttention / Tensor Core
2. **Trade-off 的艺术**: Serialization 换 KNN 精度 (xCPE 补回)；Unpooling 换插值时间 (Skip Connection 补回)
3. **Scale is all you need**: 极低显存 → 更大 Batch、更深网络，降维打击
4. **位运算复用**: 序列化代码同时服务于排序、分块、池化三个功能，零额外开销
